## 2. 반응형 UI 설계

### 2.1 브레이크포인트

| 구간 | 기준 | 레이아웃 |
| :--- | :--- | :--- |
| Mobile | `~639px` | 단일 컬럼, 세로 스택, 버튼은 full-width 우선 |
| Tablet | `640px~1023px` | 2컬럼까지 확장, 보조 패널은 하단 정렬 가능 |
| Desktop | `1024px~` | Flex/Grid 혼합, 사용자 페이지는 중심 콘텐츠 + 보조 패널, 관리자 페이지는 좌측 사이드바 + 본문 |
| Wide | `1280px~` | 본문 폭을 제한하고 정보 밀도를 유지한다. |

### 2.2 적용 원칙

- 사용자 페이지는 핵심 작업 흐름이 끊기지 않도록 `flex`와 `grid`를 조합한다.
- 입력 화면은 폼과 안내 영역을 분리하고, 결과 화면은 카드와 상세 패널을 반응형으로 재배치한다.
- 관리자 페이지는 좌측 사이드바와 우측 콘텐츠 영역을 분리한 구조를 유지한다.
- 긴 문구는 줄바꿈과 말줄임을 병행해 모바일에서도 깨지지 않게 처리한다.

## 3. 인증 및 권한 흐름

- 비로그인 사용자는 보호된 화면 접근 시 로그인 화면으로 이동한다.
- 로그인 후에는 사용자 역할에 따라 사용자 페이지 또는 관리자 페이지로 분기한다.
- 관리자 페이지는 로그인만으로 끝나지 않고 역할에 따라 접근 가능한 메뉴를 제한한다.
- 마이페이지, 작명 기록, 입력 화면은 사용자 인증이 필요하다.
- 관리자 계정 관리, 점검 설정은 상위 권한에서만 노출한다.
- 점검 모드가 활성화되면 일반 화면보다 우선해서 `MaintenanceScreen`이 표시된다.
- 관리자 앱은 `/manage/*` 별도 번들에서 동작하며, 정의되지 않은 관리자 경로는 `/manage/dashboard`로 이동한다.

## 4. LLM 상호작용 UX 설계

- 입력 단계에서 사용자는 조건을 한 번에 검토하고 제출한다.
- 생성 중에는 결과가 비어 보이지 않도록 전용 대기 화면을 제공한다.
- 결과 화면은 카드 중심으로 구성하고, 다시 생성 또는 결과 확인 동작을 분리한다.
- 후속 질의는 결과 흐름 안에서 이어지며, 별도 창 전환 없이 맥락을 유지한다.
- 비동기 요청 실패 시에는 재시도와 원인 확인이 가능한 상태 메시지를 둔다.
